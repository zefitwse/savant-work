# 任务8和任务13接口说明

本文档按 `附录_接口契约与Git工作流.md` 的统一约定整理：Kafka 消息使用
`stream_id` 作为视频流主键，消息版本字段为 `msg_version: "1.0"`，坐标字段为
`bbox_xywh = [x, y, w, h]`，REST API 优先使用 `/api/v1/` 前缀。

## 任务8：Re-ID 与跨相机追踪

### 输入事件

Re-ID worker 消费 Kafka topic：

```text
deepstream.events
```

按附录约定，推荐输入格式如下：

```json
{
  "msg_version": "1.0",
  "stream_id": "cam_001",
  "timestamp": 1714972800.123,
  "frame_id": 12045,
  "objects": [
    {
      "track_id": 42,
      "class": "person",
      "confidence": 0.92,
      "bbox_xywh": [100, 200, 150, 300],
      "roi_names": ["zone_A"],
      "attributes": {
        "helmet": true,
        "uniform": false,
        "crop_uri": "runtime/crops/cam_001/42/frame_00012045_abcd1234.jpg"
      },
      "events": ["entered_zone_A"]
    }
  ]
}
```

当前 Savant PyFunc 已经输出 `camera_id`、`bbox`、`class_name` 等旧字段。新增 Re-ID
代码会兼容旧字段，并在 Re-ID 输出/API 返回中归一为附录要求的：

```text
camera_id -> stream_id
bbox      -> bbox_xywh
class_name -> class
```

### Re-ID 处理流程

```text
deepstream.events
  -> scripts/reid_service.py
  -> 读取 attributes.crop_uri 目标裁剪图
  -> 提取 reid_feature
  -> SQLite 一小时特征缓存
  -> 跨 stream_id 相似度匹配
  -> 生成 global_id
  -> reid.events
```

如果配置了 `REID_MODEL_PATH`，会使用 ONNX Runtime 在 CPU 上提取真实 Re-ID 模型特征。
没有模型时使用 CPU fallback 特征，保证低配机器也能完成接口联调和课程报告演示。

### Re-ID 输出事件

Re-ID worker 输出 Kafka topic：

```text
reid.events
```

示例：

```json
{
  "msg_version": "1.0",
  "event_type": "reid_matched",
  "global_id": "gid-a1b2c3d4e5f6",
  "stream_id": "cam_002",
  "track_id": "17",
  "crop_uri": "runtime/crops/cam_002/17/frame_00001234_abcd1234.jpg",
  "feature_version": "onnx-osnet_x0_25",
  "feature_dimension": 512,
  "similarity": 0.86,
  "matched_record_id": 12,
  "matched_stream_id": "cam_001",
  "matched_track_id": "42",
  "timestamp": "2026-05-08T12:00:00.000+00:00"
}
```

当 crop 图暂时还没有生成时，worker 不会崩溃，会输出：

```json
{
  "msg_version": "1.0",
  "event_type": "reid_pending_crop",
  "stream_id": "cam_001",
  "track_id": "42",
  "reason": "Re-ID crop does not exist: ...",
  "timestamp": "2026-05-08T12:00:00.000+00:00"
}
```

### REST API

附录指定的以图搜图接口：

```text
POST /api/search/reid
```

同时提供符合统一前缀的接口：

```text
POST /api/v1/search/reid
```

请求示例：

```powershell
curl.exe -X POST "http://127.0.0.1:18080/api/v1/search/reid?top_k=10" `
  -F "file=@runtime/crops/cam_001/42/frame_00012045_abcd1234.jpg"
```

返回字段包含：

```text
global_id
stream_id
track_id
crop_uri
feature_version
timestamp
similarity
event
```

其他辅助接口：

```text
POST /api/v1/reid/events
POST /api/v1/reid/search/base64
GET  /api/v1/reid/tracks/{global_id}
GET  /api/v1/reid/stats
POST /api/v1/reid/prune
```

为了兼容旧联调脚本，原来的 `/reid/*` 路径仍保留，但报告和前端对接建议使用
`/api/v1/*` 或附录指定的 `/api/search/reid`。

## 任务13：解耦传输与链路追踪

### 解耦传输

视频帧链路使用 Savant 官方 ZeroMQ Adapter：

```text
Source Adapter -> ZeroMQ -> Savant Module -> ZeroMQ -> Sink
```

结构化事件链路使用 Kafka：

```text
Savant PyFunc -> deepstream.events -> Re-ID worker / 前端 / 其他消费者
```

Kafka topic：

```text
deepstream.events  目标检测与跟踪稀疏事件
reid.events        Re-ID 注册、匹配、待裁剪、失败事件
```

`docker-compose.kafka.yml` 会创建 `deepstream.events` 和 `reid.events`，并把 Kafka
加入 `deepstream_coursework_default` 网络，供 Savant、control-api、reid-worker 互相访问。

### OpenTelemetry Span

当前组件级链路追踪包括：

```text
savant.event.process_frame
savant.reid.crop_export
savant.event.build
savant.kafka.publish
reid.event.consume
reid.feature.match
FastAPI request spans
```

这些 span 能覆盖：

```text
视频帧进入 Savant
目标事件构造
crop_uri / GPU 引用字段增强
Kafka 发布
Re-ID worker 消费
特征提取与跨 stream_id 匹配
以图搜图 HTTP 请求
```

底层 DeepStream 算子级耗时仍需要更深的 runtime hook；本项目当前完成的是课程作业所需的
组件级全链路追踪。

## 本地启动命令

只启动 Kafka 与控制面：

```powershell
docker compose -f docker-compose.kafka.yml up -d
docker compose -f docker-compose.control.yml up -d --build
```

单独运行 Re-ID worker：

```powershell
python scripts/reid_service.py `
  --bootstrap-servers localhost:9092 `
  --input-topic deepstream.events `
  --output-topic reid.events `
  --db-path runtime/edge_control.db `
  --project-root .
```
